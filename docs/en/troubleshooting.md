# Troubleshooting

## When the Process Does Not Start

- Verify that the configuration file path is correctly provided as an argument
  (`reproduce <CONFIG>`).
- Verify that more than one argument is not provided.
- Verify certificate, private key, and CA certificate paths.
- If `log_path` is specified, verify that the log file can be opened.

## Configuration File Errors

- Verify that all required fields are present: `cert`, `key`, `ca_certs`,
  `giganto_ingest_srv_addr`, `giganto_name`, `kind`, and `input`.
- Verify that `kind` is not an empty string.
- If `report = true`, verify that `report_dir` is configured.

## When Connection to Giganto Fails

- Verify the IP address and port in `giganto_ingest_srv_addr`.
- Verify that `giganto_name` matches the TLS server name of the Giganto server.
- Verify that the CA certificates used for certificate validation are correct.
- Verify that the Giganto server version is 0.28.0 or later.

## When Input Data Is Not Processed

- Verify that the `input` path is correct and that the file or directory exists.
- In Directory mode, verify that `file_prefix` is not filtering out the target
  files.
- Verify that `transfer_skip_count` is not larger than the total amount of data.
- Verify that the checkpoint file generated by `last_transfer_line_suffix` does
  not contain a value larger than the available data range.
- If new data is continuously appended, verify that `polling_mode = true` is
  enabled.

## When Netflow v9 Data Is Not Parsed

- Verify that the `NETFLOW_TEMPLATES_PATH` environment variable points to the
  correct template path.
- Verify that the `netflow` feature is enabled during build.

## When Elastic Mode Cannot Retrieve Data

- Verify that all required fields in the `[elastic]` section are configured.
- Verify that `url` and `elastic_auth` (`username:password`) are correct.
- Verify that matching data exists within the configured `indices`,
  `event_codes`, and `start_time` / `end_time` range.
- Verify that the `dump_dir` path and write permissions are correct.
