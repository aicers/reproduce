# 개요

REproduce는 로그 파일이나 패킷 데이터를 읽어 이벤트로 변환한 뒤 Giganto 서버로 전송하는 프로그램입니다.
단일 로그 파일, 로그 파일이 모인 디렉터리, 또는 Elasticsearch 서버를 입력으로 사용할 수 있으며,
입력 데이터의 종류는 TOML 설정 파일에서 지정합니다.

## 주요 특징

<!-- markdownlint-disable MD007 -->
- 로그 파일 또는 패킷 데이터를 분석해 이벤트로 변환합니다.
- 입력 소스는 세 가지 중 하나로 제공됩니다.
    - 파일 모드: 단일 로그 파일을 읽어 전송
    - 디렉터리 모드: 디렉터리 안의 로그 파일들을 읽어 전송
    - Elastic 모드: Elasticsearch 서버에 질의해 받은 데이터를 전송
- 변환 결과를 Giganto 서버로 전송합니다.
- 폴링 모드로 파일에 새로 추가되는 내용을 계속 감시할 수 있습니다.
- 전송 통계 리포트를 파일로 남길 수 있습니다.
<!-- markdownlint-enable MD007 -->

## 입력 모드

- **파일 모드**: `input`에 로그 파일 경로를 지정
- **디렉터리 모드**: `input`에 디렉터리 경로를 지정
- **Elastic 모드**: `input = "elastic"`로 지정하고 `[elastic]` 섹션을 작성

> **참고**
> 입력 종류는 `input` 값으로 자동 판별됩니다. `"elastic"`이면 Elastic 모드,
> 디렉터리 경로이면 디렉터리 모드, 그 외에는 파일 모드로 동작합니다.

## 보안 전제(TLS)

REproduce는 Giganto 서버와 통신할 때 인증서 기반 TLS 설정이 필요하며, 설정 파일에
아래 항목을 지정해야 합니다.

- `cert`, `key`, `ca_certs`로 인증서, 개인 키, CA 인증서를 전달해야 합니다.
- `ca_certs`의 각 파일에는 하나 이상의 PEM 인증서(CA 번들, 풀체인 등)가 포함될 수 있습니다.

## 매뉴얼 구성

- **설치 전 준비**: 설정 파일, 인증서/키, CA 인증서 준비
- **설정**: TOML 설정 파일 작성
- **실행**: 입력 모드별 실행 방법
- **문제 해결**: 자주 발생하는 오류와 대응 방법

## 빠른 시작

1. `config.toml`을 작성
2. 인증서, 개인 키, CA 인증서를 준비
3. Elastic 모드를 사용하는 경우 `dump_dir` 경로와 쓰기 권한을 확인
4. REproduce를 실행 `reproduce <CONFIG_PATH>`
5. Giganto 서버 연결 상태를 확인
6. 로그를 통해 정상 동작 여부를 확인
